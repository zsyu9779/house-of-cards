# 项目实施质量巡检 Skill

本 Skill 用于对 House of Cards 项目进行标准化的实施质量巡检，生成巡检报告，并针对发现的问题进行修复开发。

## 何时激活

* 用户输入「进行例行巡检任务」或类似指令时
* 需要评估项目开发质量时
* 需要针对巡检问题进行修复时

## 巡检流程

### 第一步：收集项目状态信息

执行以下命令收集项目状态：

```bash
# 1. 检查测试文件存在情况
find . -name "*_test.go" -type f | sort

# 2. 运行测试并查看覆盖率
go test -v ./internal/... 2>&1

# 3. 检查依赖
go mod graph | wc -l
go mod tidy

# 4. 检查代码规范（静态分析）
go vet ./...
gofmt -l .

# 5. 统计代码行数
find . -name "*.go" -not -path "./vendor/*" | xargs wc -l | tail -1
```

### 第二步：分析核心模块

读取并分析以下关键模块的实现质量：

| 模块 | 检查要点 |
|------|---------|
| store | 测试覆盖、错误处理、SQL 注入防护 |
| whip | 测试覆盖、tick 复杂度、并发安全 |
| privy | 测试覆盖、合并逻辑、冲突处理 |
| runtime | 接口设计、多 runtime 支持 |
| chamber | worktree 操作、清理逻辑 |

根据以下模板生成巡检报告，保存到 `docs/research-reports/`:

### 第四步：执行修复开发

根据巡检报告中的问题和建议，按照优先级顺序进行修复：

#### P0 - 必须修复
1. 添加缺失的核心模块测试
2. 修复安全漏洞

#### P1 - 重要改进
1. 补充输入验证
2. 改进代码规范

#### P2 - 优化项
1. 添加注释和文档
2. 性能优化

### 修复操作规范

每个修复操作应该：
1. 创建对应的测试用例（先测试后修复）
2. 实现修复代码
3. 运行测试确保通过
4. 更新巡检报告中的状态

### 第五步：更新 Agent Context

修复完成后，更新 `.hoc/agent_context.md`：

```markdown
## 巡检任务（{DATE}）

### 本次巡检
- 巡检报告：docs/research-reports/{DATE}.md
- 发现问题：{数量} 个
- 已修复：{数量} 个
- 待处理：{数量} 个

### 本次修复内容
1. {修复1}
2. {修复2}
```

## 巡检检查清单

- [ ] 收集测试文件列表
- [ ] 运行测试套件
- [ ] 执行静态分析
- [ ] 分析核心模块代码
- [ ] 生成巡检报告
- [ ] 评估问题优先级
- [ ] 执行修复开发
- [ ] 验证修复结果
- [ ] 更新文档和上下文

## 注意事项

1. **巡检报告格式**：必须使用标准化的报告模板
2. **评分一致性**：评分标准应与上次巡检保持一致，便于对比
3. **修复验证**：每次修复后必须运行测试验证
4. **文档更新**：修复内容必须同步更新到对应文档
5. **上下文同步**：完成后必须更新 agent_context.md
